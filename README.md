## 一、游戏简介

**《盗梦空间》** 是一款 **2D 90° 顶视** 像素风格的 **不对称竞技塔防游戏**（Asymmetric Tower Defense），灵感来源于电影《盗梦空间》的多层梦境设定。

### 核心特色

- **双阵营对抗**：人类 vs 猎梦者，操作视角、胜利条件、策略体系完全不同
- **4 张风格地图**：简单（木色）、苔藓（绿地）、雪地（白雪）、炼狱（熔岩紫）
- **6 档难度 + 无尽模式**：通关炼狱后解锁无尽模式，每波 +1 猎梦者、属性 +15%
- **5 种猎梦者**：晴天娃娃 / 大头 / 孙婆婆 / 绷带 / 红裙小女孩，各具独特技能
- **20 种建筑**：基础 / 赚钱 / 防御 / 高科技 / 黑科技 / 特殊 六大类
- **零外部资源**：所有纹理与音效均 **程序化生成**，不依赖任何图片/音频文件
- **AI 智能决策**：6 个 AI 队友遵循优先级决策树，可与玩家形成"多打一"或"一打多"

### 阵营胜负条件速览

| 阵营 | 胜利条件 | 失败条件 |
|---|---|---|
| **人类** | 击杀所有猎梦者 | 玩家控制的人类死亡 |
| **猎梦者** | 击杀所有 6 个人类 | 玩家控制的猎梦者死亡 |

## 二、运行方法

### 环境要求
- **Python 3.12+**
- **Pygame 2.6+**（唯一第三方依赖）
- 操作系统：Windows / macOS / Linux

### 安装与启动

```bash
# 1. 安装唯一依赖
pip install pygame>=2.6

# 2. 启动游戏（工作目录为项目根目录）
python main.py
```

启动后默认进入主菜单，选择难度与阵营后开始游戏。窗口可拖拽边缘缩放（HUD 高度固定 56 px，游戏区按比例自适应）。

### 存档说明
游戏会自动将通关进度写入 `save.json`（项目根目录）：
- 已解锁的最高难度
- 是否通关炼狱（解锁无尽模式）
- 无尽模式最高波数

> ⚠️ `save.json` 在 `.gitignore` 中被忽略，不会污染仓库。

## 四、两种阵营

### 人类阵营（MODE_HUMAN）
- **玩家控制** 1 个人类（蓝色），其余 5 个为 AI 队友
- 目标：**击杀所有猎梦者**
- 玩法核心：
  - 在 30 秒 **安全期** 内规划房间布局
  - 人类走入房间门后即"上床"——开始产金币（床升级 1 → 32 金币/秒）
  - 在房间内建造 **门/炮塔/维修台/矿/高科技建筑** 构筑防线
  - 借助 AI 队友的协同决策，应对越来越强的猎梦者

### 猎梦者阵营（MODE_HUNTER）
- **玩家控制** 1 个猎梦者，其余 5 个人类全为 AI
- 目标：**击杀所有 6 个人类**
- 玩法核心：
  - 出生在地图边缘的 **泉水出生点**（自动回血）
  - 通过 A* 寻路接近目标房间，**破门** 后进入房间秒杀
  - 累计伤害达到阈值自动 **升级**（Lv1→Lv5，HP/ATK/攻速递增）
  - 在低血量时可 **逃跑回泉水** 回血（自动判定）

## 六、四张地图

所有地图采用统一的 **33 列 × 26 行** 矩阵描述（JSON 配置 + 走廊网格），地图风格仅通过 **调色板** 区分，加载逻辑完全统一。

| 地图 | 文件 | 走廊风格 | 调色板特点 |
|------|------|----------|------------|
| 简单 (Easy) | `world/config/map_easy.json` | 木色 | 暖棕走廊 + 米黄房间 |
| 苔藓 (Normal) | `world/config/map_normal.json` | 苔藓 | 绿色草地 + 暗绿墙 |
| 雪地 (Snow) | `world/config/map_snow.json` | 雪地 | 白色走廊 + 蓝白墙 |
| 炼狱 (Hell) | `world/config/map_hell.json` | 熔岩 | 暗紫走廊 + 红黑墙 |

### 矩阵值定义（统一）

| 值 | 含义 | 备注 |
|----|------|------|
| `0` | 走廊 | 可通行 |
| `1` | 房间内部 | 人类可通行；猎梦者仅破门后可达 |
| `2` | 墙体 | 不可通行 |
| `3` | 房门（蓝格） | 初始门位置 |
| `4` | 红格（进门箭头） | 人类踏入触发 **门滑入动画** |
| `5` | 出生点 | 猎梦者出生，站立可回血 |

### 门滑入动画（4 张地图通用）
- 每个房间门（蓝格 `3`）有 1 个相邻红格（`4`）
- 人类踏入红格时，门 **从蓝格平滑滑入红格**（0.35 秒）
- 蓝格变墙，红格变门；动画结束后红格恢复为可通行
- **视觉上**门会"滑过来挡住"走廊——这是项目的代表性动画细节

### 猎梦者出生点
- 分布在地图四边凸出的走廊格（多个分散点，避免集中刷新）
- 出生点等同走廊，可通行 + 站立回血

## 七、核心机制

### 1. 房间 + 床 经济
- 每个房间有 1 个 **床位**，人类走入即"上床"
- 上床后每秒产出金币（按床等级 1/2/4/8/16/32）
- 玩家在床状态下才能建造 / 升级建筑

### 2. 门 3 材质 15 级
- **木门** (Lv 1-5)：HP 500 → 3500
- **铁门** (Lv 6-10)：HP 5000 → 17000
- **金门** (Lv 11-15)：HP 22000 → 52000
- 升级费用分段：木 0→500，铁 800→3500，金 5000→18000
- 门 HP 受攻击时动态影响 A* 寻路可达性

### 3. 炮塔自动攻击
- 4 级，射程 5→8 格，DPS 12→50
- 每 0.7~1.5 秒发射一颗子弹（追踪型，速度 360 px/秒）
- **可达性约束**：猎梦者须在走廊/泉水 或 门已破的房间内

### 4. 维修台累积回血
- 5 级，回血 70→420 HP/秒
- 使用 **小数累积器** `_repair_accum`，避免 dt 整数丢失

### 5. 屠龙刀斩击动画（v1.7 新增）
按 `F` 键时：
1. **飞向** 猎梦者（0.25 秒）
2. **砍击** 动作（0.15 秒），造成 20% 最大 HP 伤害
3. **飞回** 原位（0.30 秒）
- 无距离限制，自动优先砍正在攻击你房间门的猎梦者
- 冷却 15 秒

### 6. 5 种猎梦者 + 5 级成长

**升级阈值**（累计伤害）：`0 / 350 / 1200 / 3500 / 8000`

| Lv | HP | ATK | 攻速(次/秒) | 移速(格/秒) |
|----|----|----|-----------|------------|
| 1 | 500 | 10 | 0.7 | 3.5 |
| 2 | 800 | 15 | 0.9 | 3.5 |
| 3 | 1200 | 24 | 1.1 | 3.5 |
| 4 | 1700 | 35 | 1.4 | 3.8 |
| 5 | 2500 | 52 | 1.7 | 4.2 |

升级额外加成：**每级 +5% 攻击力与攻速**；升级时回满 HP，解除逃跑状态。
升级到 **Lv3** 时 BGM 自动切换为紧张曲目。

### 7. 逃跑机制
- 阈值：`HP ≤ 20% 最大血量`
- 触发后：取消当前目标 → 寻路回泉水 → 站立回血
- HP 满血后自动取消逃跑状态

### 8. 难度解锁与存档
- 6 档难度按顺序解锁
- 通关 **炼狱** 后解锁无尽模式
- 通关信息写入 `save.json`（已加入 `.gitignore`）

- **泉水回血**：在出生点站立回血 20 HP/秒

### 7 大技能效果
- `shield_active` 能量罩激活中
- `berserk_active` 狂暴
- `stunned_timer` 诱捕网定身

- `atk_speed_mult` 冰箱减速（×0.8）
- `turret_disabled` 红裙小女孩炮停
- `bandage_healing` 绷带回血

## 十、AI 行为设计

### AI 人类决策树（`_ai_human_decide`，每 3-5 秒决策一次）

```
1. 升级床（首选，房间经济基础）
2. 升级门（门等级 < 床等级时）
3. 防御核心：炮塔（房间 1-3 座）
4. 维修台（炮塔存在后，0-2 座）
5. 经济：游戏机（房间 0-2 台，给高科技供电）
6. 矿坑（有电时 50% 概率建铜矿）
7. 高科技/黑科技（视电力与资源）：
   - 冰箱 → 能量罩 → 诱捕网 → 断头台
8. 特殊道具（随机选择模拟真人偏好）：
   - 镜子 (40%) / 小草 (50%) / 一盆小草 (40%)
   - 大蒜 (35%) / 蛤蟆 / 屠龙刀
9. 升级炮塔
```

> AI 严格遵守 **电力门槛**——游戏机未建时不会乱建耗电建筑。

### AI 猎梦者决策（`_ai_hunter_decide`）

- **目标选择**：在所有可破门房间中，优先选择 **有活人 + 门 HP 最低** 的房间
- **路径规划**：A* 寻路 → 破门 → 追击房间内人类 → 秒杀
- **逃跑判定**：HP ≤ 20% 时寻路回泉水
- **特殊种类**：
  - 晴天娃娃 / 大头：攻击时尝试触发狂暴
  - 孙婆婆：升级时尝试召唤分身
  - 绷带：HP 低时进入自动回血状态
  - 红裙小女孩：攻击时尝试触发炮停

### 移动系统

- 网格坐标 `(grid_col, grid_row)` + 像素插值 `(px, py)`
- 移动节流：`1.0 / HUMAN_SPEED` 秒/格
- 像素速度：`HUMAN_SPEED × TILE_SIZE = 4 × 32 = 128 px/秒`
- 子弹速度：`BULLET_SPEED = 360 px/秒`

```
└── README.md              # 本文件
```

### 架构原则

- **单向依赖**：`core.config` ← 业务模块 ← `main`，无循环引用
- **数据/逻辑分离**：`entities/`（纯数据类）+ `systems/`（纯逻辑函数）
- **表现层独立**：`ui/` 和 `graphics/` 接收 `GameState` 引用进行绘制
- **2D 顶视渲染**：始终保持正交俯瞰，不引入 3D/等距/伪 3D

## 十二、技术栈与亮点

### 技术栈

| 类别 | 选型 | 备注 |
|------|------|------|
| 编程语言 | Python 3.12 | 仅使用标准库 `random` / `heapq` / `math` / `sys` / `json` |
| 游戏框架 | Pygame 2.6 | 唯一第三方依赖，承担渲染、事件、字体与 Surface |
| 中文字体 | `simhei`（系统黑体） | 缺失时自动回退到 `pygame.font.Font(None, …)` |
| 操作系统 | Windows / macOS / Linux | 已用 `pygame.RESIZABLE` 适配窗口缩放 |

### 技术亮点

#### 1. 零图片资源
- 所有纹理（角色/建筑/门/光效/地形）由 `pygame.draw.*` 程序化生成
- `graphics/renderer.py` 提供 22+ 种像素精灵绘制函数（`draw_pixel_texture` 缓存机制）
- 角色 / 床 / 炮塔 / 维修台 / 门 等都有 **多级升级精灵**

#### 2. 零音频资源
- SFX 全部由 `struct + math + wave` 程序化生成（点击/炮击/中弹/死亡/升级/屠龙刀）
- BGM 优先加载 `assets/audio/bgm/*.ogg`，缺失时回退程序化生成的环境音
- `core/audio.py` 实现 BGM/SFX 双通道音量独立控制

#### 3. A* 寻路系统
- 启发式：**曼哈顿距离** `abs(a[0]-b[0]) + abs(a[1]-b[1])`
- 数据结构：`heapq` 优先队列
- 实体通行规则（动态门 HP 影响可达性）：

| 实体类型 | 走廊 | 泉水 | 房间 | 门（完好） | 门（破） | 床 |
|---------|:----:|:----:|:----:|:---------:|:-------:|:--:|
| `hunter` | ✓ | ✓ | 仅破门 | ✗ | ✓ | ✗ |
| `human_wandering` | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| `human_bed` | ✗ | ✗ | ✗ | ✗ | ✗ | ✗ |

#### 4. 状态机
```
PHASE_MENU → PHASE_INIT → PHASE_SAFE → PHASE_PLAYING → PHASE_VICTORY / PHASE_DEFEAT
                  │            │
                  └─── ESC ────┘─── P (暂停) ───┐
                                                 ↓
                                          (渲染继续, 逻辑暂停)
```

#### 5. 渲染管线
- 屏幕 1280×720 → HUD 56 px → 游戏区按比例缩放
- 13×13 格视口 → viewport 跟随 → 屏幕缩放
- **逐像素绘制**，无 sprite 缩放失真

#### 6. 持久化
- `save.json` 自动保存通关进度
- 通关炼狱 → 全部难度解锁
- 无尽模式最高波数记录
