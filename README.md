# 《盗梦空间》Dream Stealing

> 基于 Python + Pygame 的像素风格**双阵营对抗塔防**游戏。
> 玩家可选"人类"或"猎梦者"两个阵营，体验完全不同的目标与策略。

![游戏截图](简单地图像素版.png)

---

## 快速开始

```bash
# 唯一依赖
pip install pygame>=2.6

# 启动
python main.py
```

支持 Windows / macOS / Linux。窗口可缩放，分辨率自适应。

---

## 操作

| 按键 | 功能 |
|------|------|
| `W` `A` `S` `D` / 方向键 | 移动角色（4 格/秒） |
| 鼠标左键 | 建造 / 升级 / 切换跟随目标（点击 HUD 头像） |
| 鼠标拖拽 | 拖动摄像机视野 |
| `F` | 激活**屠龙刀**（无距离限制，优先砍正在攻击你房间门的猎梦者） |
| `P` | 暂停 / 继续 |
| `ESC` | 返回主菜单 |
| `SPACE` | 结算界面确认 |

## 两种模式

| 阵营 | 目标 | 你 | 盟友 |
|------|------|----|------|
| **人类** (`MODE_HUMAN`) | 击杀猎梦者 | 玩家 + 5 个 AI 人类 | 全部队友是 AI |
| **猎梦者** (`MODE_HUNTER`) | 消灭所有人类 | 玩家 + 1+ 个 AI 猎梦者 | 6 个人类全是 AI |

## 6 档难度 + 无尽模式

简单 → 普通 → 困难 → 噩梦 → 地狱 → 炼狱 → **无尽**（通关炼狱后解锁）
- 每关使用不同猎梦者类型/数量
- 无尽模式每波 +1 猎梦者，属性 +15%/波

## 核心机制

### 4 张地图，统一 33×39 矩阵
所有地图共用同一套 6 值矩阵（0=走廊, 1=房间, 2=墙, 3=门, 4=进门箭头, 5=出生点），加载逻辑完全统一。

- 简单 (Easy) — 木色走廊
- 苔藓 (Normal) — 绿色草地
- 雪地 (Snow) — 白色雪地
- 炼狱 (Hell) — 熔岩紫

### 门滑入动画（**4 张地图通用**）
- 每个房间门（蓝格 3）有 1 个相邻红格（进门箭头 4）
- 人类踏入红格时，门**从蓝格平滑滑入红格**（0.35 秒）
- 蓝格变墙，红格变门，动画结束后红格恢复为可通行

### 猎梦者 5 级成长
- 5 种猎梦者：**晴天娃娃 / 大头 / 孙婆婆 / 绷带 / 红裙小女孩**，各具独特技能
  - 晴天娃娃/大头：**狂暴**（攻 ×1.4, 速 ×1.3，持续 10s）
  - 孙婆婆：**分身**（升级时召唤 30% HP 的分身，合体后主体永久 +5 攻击）
  - 绷带：**自动回血**（HP<30% 时 30 HP/s 回血到 50%）
  - 红裙小女孩：**炮停**（全图炮塔停火 6s，冷却 40s）
- 累计伤害达到阈值自动升级（Lv1→Lv5，HP/ATK/攻速递增）
- 升级到 Lv3 时 BGM 切换为紧张曲目

### 20 种建筑，多级升级
- **基础**：门（15 级，木/铁/金 3 材质）× 床（6 级）× 游戏机（5 级）
- **防御**：炮塔（4 级）× 维修台（5 级）
- **赚钱**：铜/银/金/钻石矿（4 级，耗电）
- **黑科技**：诱捕网 / 断头台 / 镜子 / 大蒜 / 蛤蟆 / 屠龙刀 / 冰箱 / 能量罩 / 小草
- **屠龙刀**：按 F 键手动斩 20% 最大 HP 伤害（带飞刀动画）

### AI 行为
- **AI 人类**：每 3-5 秒决策一次（升级床→升级门→造炮塔→造维修台→升级炮塔）
- **AI 猎梦者**：优先攻击有活人 + 门最低的房间，破门后追击；绷带靠自回血，其余靠玩家击杀节奏

## 项目结构

```
Dream_Stealing/
├── main.py              # 游戏入口、主循环、状态机
├── core/                # 配置/状态/音频/存档
├── entities/            # Human / DreamHunter / Building / Bullet / Tongue
├── systems/             # 经济/战斗/AI/移动/胜负/门滑入动画
├── graphics/            # 程序化像素纹理 + 渲染
├── ui/                  # 主菜单/HUD/摄像机/输入路由
├── world/
│   ├── map_data.py      # 矩阵加载 + A* 寻路
│   └── config/          # 4 张地图的 33×39 JSON
├── tools/               # 维护脚本（地图生成、烟雾测试等）
├── assets/audio/bgm/    # BGM 文件（可选，缺失时程序化生成）
├── *.png                # 4 张地图像素版（参考图）
└── README.md
```

**架构原则**：数据/逻辑分离（`entities/` + `systems/`），表现层独立（`ui/` + `graphics/`）。
- `core/config.py` → 全局常量（数据维度）
- `systems/systems.py` → 核心玩法逻辑（约 1300 行）
- `graphics/renderer.py` → 全部程序化像素绘制（无任何外部图片资源）

---

## 技术栈

- **Python 3.12** + **Pygame 2.6**（唯一第三方依赖）
- **零图片资源**：所有纹理（角色/建筑/门/光效）由 `pygame.draw.*` 程序化生成
- **零音频资源**：SFX 全部由 `struct + math` 程序化生成，BGM 可选文件
- **A\* 寻路**：曼哈顿距离 + `heapq` 优先队列
- **状态机**：`MENU → SAFE → PLAYING → VICTORY/DEFEAT`
- **渲染管线**：13×13 格视口 → 416×416 viewport → 屏幕缩放
- **代码规模**：约 **10,500+ 行**，13 个核心模块
