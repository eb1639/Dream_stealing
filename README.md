## 这是什么

《盗梦空间》是一款 2D 顶视像素塔防游戏，灵感来自同名电影的多层梦境设定。

游戏最大的特点是 **两个阵营完全不对称**——玩家既可以扮演 1 名人类 + 5 名 AI 队友在 9 个房间里建造防御，也可以反过来扮演 1 名猎梦者，独自破门、潜入、猎杀全部 6 个人类。两条路线拥有完全不同的操作视角、胜利条件和经济系统。

地图共 4 套主题（简单 / 苔藓 / 雪地 / 炼狱），难度从 6 档主线递进到无尽模式。

## 特性

- **双阵营不对称对抗** —— 人类 vs 猎梦者，操作、目标、策略体系完全分离
- **零外部资源依赖** —— 所有像素精灵与音效均由 Python 代码程序化生成，仓库里没有任何图片 / 音频二进制
- **4 张主题地图 + 6 档难度 + 无尽模式** —— 统一 33×26 矩阵描述，仅调色板差异
- **5 种猎梦者 + 20 种建筑** —— 全部由数据表驱动，加新内容只需在 `config.py` 加行
- **A* 寻路 + 动态门** —— 门的 HP 实时影响寻路可达性
- **AI 决策树** —— 5 名 AI 队友严格按 `床 → 门 → 炮塔 → 维修台 → 矿 → 高科技` 优先级决策
- **存档系统** —— 通关进度写入 `save.json`，已加入 `.gitignore`

## 快速开始

```
# 1. 安装唯一第三方依赖
pip install pygame>=2.6

# 2. 启动游戏（请确保工作目录为项目根目录）
python main.py
```

启动后进入主菜单，选择难度与阵营即可开始。窗口可拖拽边缘缩放（HUD 高度固定 56 px，游戏区按比例自适应）。

> 环境要求：Python 3.12+ · Pygame 2.6+ · Windows / macOS / Linux

## 架构设计

### 模块依赖（单向，无循环）

```
core.config
    ↓
entities  ←  world  ←  systems
                    ↖   ↙
                  graphics / ui
                      ↓
                    main.py
```

- `core.config` 是**唯一数据源**——所有建筑、猎梦者、难度常量都集中在这一个文件
- `entities/` 是**纯数据类**（dataclass），不含任何逻辑
- `systems/` 是**纯逻辑函数**，接收 `GameState` 并就地修改
- `ui/` 和 `graphics/` 是**表现层**，只读取 `GameState` 并绘制
- `main.py` 串联以上所有模块，驱动主循环

### 关键技术亮点

1. **零图片 / 零音频资源**
   全部 22+ 种像素精灵（角色、门 3 材质、床、炮塔、维修台、矿、屠龙刀、能量罩、飘字等）均由 `pygame.draw.*` 组合绘制，并通过 `Surface` 缓存避免重复计算。SFX 全部由 `struct + math + wave` 程序化合成；BGM 优先加载 `assets/audio/bgm/*.ogg`，缺失时回退到程序化生成的环境音。

2. **A* 寻路 + 动态门**
   启发式为曼哈顿距离，结构用 `heapq` 优先队列。门是路径上的**动态障碍**——门 HP > 0 时阻挡猎梦者，HP = 0 时自动变成普通走廊。AI 队友和猎梦者共用同一套 A*，但通行规则不同（猎梦者需破门才能进入房间）。

3. **状态机驱动游戏循环**
   ```
   PHASE_MENU → PHASE_INIT → PHASE_SAFE → PHASE_PLAYING → PHASE_VICTORY / PHASE_DEFEAT
   ```
   固定 60 FPS 主循环，dt 钳制到 0.1 秒以内防止卡顿后瞬移，暂停时跳过更新但不跳过渲染。

4. **配置驱动的可扩展性**
   4 张地图、20 种建筑、5 种猎梦者、6 档难度全部由 `core/config.py` 数据表驱动。新增一个建筑只需在表里加一行 + 在 `graphics/renderer.py` 加一个绘制函数，无需改动其他模块。

5. **门滑入动画**
   人类踏入房间门前的红格时，门从蓝格**平滑滑入**红格（0.35 秒），蓝格变墙、红格变门——是项目里最具辨识度的视觉细节。
